\begin{abstract}
RNA 剪接是连接基因组序列、转录本结构与功能变异的重要分子过程。经典剪接位点通常包含 donor 侧的 GT 和 acceptor 侧的 AG 等局部共识序列，但是，真实剪接调控并不等价于对短 motif 的简单匹配：真实基因组中存在大量携带 GT/AG 却不被注释为剪接位点的诱饵序列，近剪接位点单核苷酸变异也可能改变剪接信号。围绕这一问题，本文基于 GRCh38、GENCODE v49 和 ClinVar 构建了一个小规模但真实的计算生物学 benchmark：首先建立 donor、acceptor 与 GT/AG hard-negative 三分类模型；随后比较不同上下文窗口下的识别性能；最后在 held-out chromosomes 上的 ClinVar pathogenic/likely pathogenic 与 benign/likely benign SNV 中评估变异效应排序，并用 exact-distance-matched 子集诊断距离混杂。结果表明，真实 GT/AG hard-negative 是有效压力测试；在 ClinVar 小样本上，RNA-FM frozen encoder + MLP 取得最高 AUROC/AUPRC，但 distance-matched 结果明显下降，说明近剪接距离仍是重要混杂因素。
\end{abstract}

\begin{abstract*}
RNA splicing links genomic sequence, transcript structure, and functional variants. Canonical splice sites often contain donor-side GT and acceptor-side AG motifs, but real genomes also contain many motif-matched decoys. This project builds a small real benchmark from GRCh38, GENCODE v49, and ClinVar: splice-site three-class classification, multi-scale context analysis with GT/AG hard negatives, and ClinVar pathogenic-vs-benign variant effect ranking with distance-matched diagnostics.
\end{abstract*}

\chapter{引言}

RNA 剪接是从基因组编码到转录本功能的关键中间层，决定了前体 mRNA 中外显子与内含子的连接方式。剪接位点识别通常依赖 donor 位点附近的 GT、acceptor 位点附近的 AG，以及多聚嘧啶区、分支点和上下游调控元件等信息。

已有研究已经从经典剪接信号、最大熵模型和深度神经网络等角度证明，剪接位点附近的序列包含强预测信号\cite{black2003,yeo2004,jaganathan2019}。近年来，SpliceAI、Pangolin、MMSplice 等工具进一步将序列模型用于变异效应预测\cite{jaganathan2019,zeng2022pangolin,cheng2019}；RNA-FM、RNABERT 等 RNA language model 则提供了从大规模无标注 RNA 序列中学习表示的思路\cite{chen2022rnafm,akiyama2022rnabert}。这些工作共同提示，剪接预测不应只被理解为简单的二核苷酸查找。

本项目关注以下问题：剪接模型究竟是在学习真实基因组中的剪接上下文，还是仅仅在匹配短 motif？

这一问题在变异效应预测中同样重要，一个单核苷酸变异可能破坏原有 donor 或 acceptor，也可能在非剪接区域制造新的 cryptic splice signal，若模型只能识别局部 GT/AG，它可能在 hard-negative 上产生大量误报，也可能难以解释变异造成的剪接信号变化。

因此，单独报告三分类准确率是不够的，更完整的分析应当包括：1.上下文消融，2.困难负例，3.非经典 motif，4.变异 delta score 和可解释性分析。

对此，我们将全文组织为三个研究问题。

\begin{itemize}
  \item RQ1：在真实 GENCODE/GRCh38 小样本条件下，模型能否区分 donor、acceptor 与 GT/AG hard-negative？
  \item RQ2：真实上下文窗口长度是否影响模型拒绝 motif-matched negative 的能力？
  \item RQ3：剪接识别信号能否转化为真实 ClinVar pathogenic-vs-benign 变异效应排序？
\end{itemize}

实验一回答 RQ1。实验二主要通过多尺度上下文和 GT/AG hard-negative 回答 RQ2，rare motif 只作为补充 synthetic stress test。实验三通过真实 ClinVar SNV、distance-matched 诊断和解释性图件回答 RQ3。

\chapter{数据与方法}

\section{数据来源、构造与小样本设定}

项目主实验使用真实 GRCh38/GENCODE v49 splice-site benchmark，而不是 synthetic 主数据集。输入资源包括 \codepath{data/raw/genome.fa}、\codepath{data/raw/gencode.gtf} 和 ClinVar VCF；\codepath{reports/data_qc.md} 记录了 GRCh38、GENCODE v49、Ensembl 115、ClinVar fileDate 2026-05-30、随机种子 42 和构建命令。

真实 splice-site 数据由 \codepath{src/data/build_splice_site_dataset.py} 生成。donor 定义为 intron 5' boundary，acceptor 定义为 intron 3' boundary；负链转录本先按 transcript orientation 反向互补。positive 位点经过 canonical GT/AG 过滤。non-splice 类不是随机 easy negative，而是在真实基因组中抽取的不落在 GTF 注释剪接位点上的 GT/AG motif-matched decoy，因此全部属于 hard-negative。

全量采样比例为 donor / acceptor / hard-negative = 1000 / 1000 / 1000。主 split 采用 chromosome holdout：chr1--chr16 为 train，chr17--chr18 为 valid，chr19--chr22 与 chrX 为 test。QC 检查显示 train/valid/test 的 sample id、chromosome 与 gene id 互斥。

主实验默认使用 pm200，即中心位点左右各 200 bp、总长 401 bp；实验二额外使用 pm50、pm100、pm400。表 \ref{tab:data-split} 给出 pm200 split 规模。

\section{变异数据构造}

实验三使用 \codepath{src/data/build_clinvar_variant_dataset.py} 构造真实 ClinVar SNV benchmark。positive 为 pathogenic / likely pathogenic 且 splice-related 或 near-splice 的记录；negative 为 benign / likely benign 且 near-splice 的记录。所有变异限制为 SNV，REF 与 GRCh38 校验，负链 REF/ALT 转为 transcript orientation；target class 由最近 donor/acceptor 决定。

\codepath{data/experiment_3/clinvar_splicing_variants.csv} 包含 500 条变异，其中 splice-altering 250 条、neutral 250 条；ClinVar 变异限制在 held-out test chromosomes，并排除实验一/二已经采样的 genes。QC 显示 pathogenic/splice-altering 变异距离注释剪接位点更近：median distance 为 9.5 bp，而 neutral 为 17 bp；distance-only baseline 在 full ClinVar 上达到 AUROC 0.6475、AUPRC 0.6917。因此，本文同时导出 exact-distance-matched 子集 \codepath{data/experiment_3/clinvar_splicing_variants_distance_matched.csv}，包含 326 条变异，正负样本 163/163，distance-only AUROC/AUPRC 均为 0.5。

\section{训练设置与复现细节}

所有可训练模型使用固定随机种子 42。实验一的 CNN baseline 使用 PyTorch Conv1D：三个卷积分支的 kernel size 分别为 3、5、9；每个分支包含 Conv1D、BatchNorm、ReLU、第二层 Conv1D、BatchNorm、ReLU 和 pooling；分类头使用 dropout、128 维全连接层、ReLU、dropout 与三分类输出层。CNN 训练 5 个 epoch，batch size 为 256，优化器为 AdamW，学习率 \(1\times10^{-3}\)，weight decay 为 \(1\times10^{-4}\)，损失函数为带类别权重的 cross entropy。当前实现报告 validation 指标，但按 validation Macro-F1 保存最佳 epoch；seed=42 的最佳 epoch 为第 5 轮。

RNA-FM/RNABERT 路线使用本地权重加载 \codepath{multimolecule} 模型，将 DNA 序列中的 T 转换为 U 后输入 tokenizer，冻结 encoder 并对 last hidden states 进行 attention-mask mean pooling。pooled embedding 与 engineered splice signal 特征拼接后输入 scikit-learn \codepath{LogisticRegression}，其中 \codepath{class_weight="balanced"}、\codepath{solver="lbfgs"}、\codepath{C=1.5}、\codepath{max_iter=1200}。实验三的变异效应分数由同一三分类模型对 WT/Mut 序列的 donor/acceptor/non-splice 概率变化计算得到。本文额外对实验一运行 42/43/44 三个种子的补充汇总，结果写入 \codepath{results/experiment_1/tables/experiment_1_multiseed_summary.csv}；CNN 的 Macro-F1 mean \(\pm\) std 为 \(0.8273 \pm 0.0124\)，RNA-FM/RNABERT 在当前确定性 frozen-encoder 设置下标准差为 0。

\section{评估指标}

三分类任务：评估 Accuracy、Macro-F1、macro one-vs-rest AUROC 和 macro one-vs-rest AUPRC，并单独统计 hard-negative FPR（hard-negative FPR 的分母限定为测试集中 \codepath{label == 2} 且 \codepath{negative_type} 包含 hard 的样本）。若将 splice-site positive 与 non-splice negative 合并为二分类，数据全量 positive base rate 为 \(2000/3000=0.667\)；本文表中的多分类 AUPRC 是 macro one-vs-rest 指标。

变异效应任务：将 ClinVar splice-altering 与 neutral 作为二分类排序问题，主表评估 AUROC 与 AUPRC。ClinVar benchmark 为 250/250 balanced，因此 AUPRC 随机基准为 0.5。本文同时报告 exact-distance-matched 子集上的 AUROC/AUPRC，用于诊断模型是否主要利用“距离剪接位点更近”这一捷径。

\chapter{实验一：剪接位点三分类}

\section{实验设计}

实验一将序列分为 donor、acceptor 和 non-splice 三类，建立上下文分析和变异效应预测的基础分类器。

模型类别包括 CNN baseline、RNA-FM frozen encoder + MLP 和 RNABERT frozen encoder + MLP。RNA-FM/RNABERT 均使用本地权重。

训练使用 2339 条样本，验证使用 230 条样本，测试使用 431 条样本。

\section{结果}

表 \ref{tab:exp1-test} 展示测试集结果，与基线进行对比。

CNN baseline 与 RNABERT frozen encoder + MLP 的 Macro-F1 接近，分别为 0.8248 和 0.8199；

三类模型在真实 GT/AG hard-negative 上的 FPR 都约为 0.32，说明仅凭当前小样本训练仍难以完全拒绝真实 motif-matched decoy。

\section{小结}

实验一说明，在小样本条件下，序列模型能够完成基础剪接位点识别，但三类模型在 GT/AG hard-negative 上仍有约三成误报，说明真实诱饵位点比简单 motif 匹配更难处理。

这个问题引出实验二：剪接识别是否需要更大范围的上下文证据？

\chapter{实验二：多尺度上下文与困难负例}

\section{实验设计}

实验二围绕三个问题展开：

第一，改变上下文窗口大小 （pm50，pm100，pm200，pm400），观察模型性能是否随上下文变化。

第二，构造 GT/AG hard-negative，检验模型是否把局部 motif 误判为真实剪接位点。

第三，构造 rare motif 压力测试，观察真实模型对非经典中心二核苷酸改写的敏感性。

\section{多尺度上下文结果}

表 \ref{tab:exp2-context} 给出了不同窗口下的 Macro-F1 与 hard-negative FPR。

\section{Hard-negative 与 rare motif}

在 155 条 hard-negative 测试样本上：

CNN baseline 产生了 55 个 false positives，FPR 为 0.3548；

RNA-FM frozen encoder + MLP 产生了 48 个 false positives，FPR 为 0.3097；

RNABERT frozen encoder + MLP 产生了 51 个 false positives，FPR 为 0.3290。

由于 hard-negative 分母为 155 条，上述比例仍需按小样本估计解读。按 Wilson 95\% 置信区间，CNN baseline 为 0.2839--0.4328，RNA-FM frozen encoder + MLP 为 0.2422--0.3863，RNABERT frozen encoder + MLP 为 0.2600--0.4064。三者区间重叠较多，因此当前结果不支持“某一模型显著优于其他模型”的强结论；更稳健的结论是，真实 GT/AG hard-negative 对三类模型都构成压力。

这一现象也提示，rare motif case study 不应被解释为真实罕见剪接位点 benchmark。当前 rare motif 是在 synthetic 序列上替换中心二核苷酸得到的压力测试，其价值在于暴露不同特征路线对 motif 改写的敏感性，而不是估计真实 GC-AG 或 AT-AC 剪接事件的总体召回率。

\section{小结}

实验二将实验一的分类结果推进到机制层面：局部 motif 是必要线索，但不能保证分类正确。

实验二主要由真实 GT/AG hard-negative 压力测试支撑。更准确地说，本文能支持的结论是：在真实基因组中，GT/AG motif 本身远非充分判据；当前小样本模型仍会在 motif-matched negative 上产生可观误报。Rare motif 仅作为 synthetic control，用于观察模型对非经典二核苷酸改写的敏感性。

实验三完成了从剪接识别到真实 ClinVar 变异效应排序的进一步深入，并加入了 SpliceAI、Pangolin、MMSplice 与 MaxEntScan 的工具推理结果。Full ClinVar 上 RNA-FM frozen encoder + MLP 排名最高，但 distance-matched 子集上的整体下降说明近剪接距离是重要混杂因素。因此，本文能支持的是“当前流程可在真实 ClinVar 小样本上形成可解释排序诊断”，而不是“完成了临床级变异泛化 benchmark”。

\chapter{讨论}

\section{三个实验之间的逻辑关系}

本文的三个实验构建了一条递进链路。实验一在真实 GENCODE/GRCh38 数据上建立 donor、acceptor 与 GT/AG hard-negative 三分类；实验二检验上下文窗口和 hard-negative 对模型的压力；实验三将识别信号转化为真实 ClinVar pathogenic-vs-benign 排序，并用 distance-matched 子集诊断混杂。

若没有实验二，实验一的总体 Macro-F1 可能掩盖 motif-matched negative 上的误报；若没有实验三，剪接识别只能停留在分类任务，无法连接到变异解释。三个实验合在一起，使本文从“能否识别剪接位点”推进到“这种识别是否可用于真实变异排序，以及边界在哪里”。

\section{关键现象解释}

第一，真实 GT/AG hard-negative 是比总体 Accuracy 更敏感的诊断指标。当前 non-splice 样本全部携带 GT/AG decoy 且不落在 GTF 注释剪接位点上，因此模型不能通过短 motif 查找解决任务。实验一和实验二中三类模型仍有约三成 hard-negative FPR，说明小样本条件下真实诱饵位点仍难以完全拒绝。

第二，上下文窗口的作用是非单调的。更长窗口可能提供更多调控信息，也可能引入更多噪声和分布差异。实验二中 RNABERT 在 pm400 上表现最好，但 CNN 在 pm50 上 Macro-F1 更高；因此本文不把“上下文越长越好”作为结论，而只说明上下文尺度需要和模型表示、样本规模共同评估。

第三，变异效应排序必须处理距离混杂。ClinVar splice-altering 变异更靠近注释剪接位点，这既符合生物学直觉，也会使模型通过“越近越可能致病”获得较高分数。Full ClinVar 和 exact-distance-matched 子集之间的差异说明，真实小样本 benchmark 中仅报告一个 AUROC/AUPRC 是不够的。

第四，外部工具的结果不能简单排行成“谁更强”。SpliceAI、Pangolin、MMSplice 和 MaxEntScan 的输入假设、输出量纲与训练目标并不相同；本文只是把它们作为真实工具参照接入同一 ClinVar 小样本排序流程。MMSplice 在当前表上输出常数型分数，说明该集成或输入设置没有提供有效区分度，应作为负结果如实报告，而不是过度解释。

\section{限制}

首先，splice-site 数据只有 3000 条，ClinVar 只有 500 条，本文已对实验一补充 3 个随机种子的 mean \(\pm\) std，但实验二和实验三仍以 seed=42 为主。其次，当前 split 采用 chromosome holdout，并检查 gene overlap；QC 进一步完成了跨 split 完整窗口重复、中心 161 bp 重复和高 9-mer Jaccard 近重复审计，但这仍不能替代 BLAST/CD-HIT/MMseqs 等 alignment-level paralog clustering。第三，ClinVar benchmark 限于 near-splice SNV，不覆盖 indel、深内含子变异、结构变异或组织特异性剪接事件。第四，distance-matched 子集只能诊断并部分缓解距离混杂，不能替代更大规模、严格匹配的临床 benchmark。

后续工作可扩展到更大规模 GENCODE/ClinVar/SpliceVarDB 数据、同源聚类去重、多随机种子、按 validation 选择 checkpoint，以及真实 GTEx/sQTL 表达数据。只有在这些条件满足后，才适合讨论全基因组泛化、组织特异性剪接或临床级变异解释能力。

\chapter{结论}

本项目构建了真实序列分类、机制压力测试和 ClinVar 变异解释的完整链路。

三个实验共同说明：

第一，在真实 GENCODE/GRCh38 小样本条件下，可以建立 donor、acceptor、GT/AG hard-negative 三分类模型，但真实 motif-matched negative 仍会带来约三成误报；

第二，多尺度上下文分析表明，窗口长度对模型表现有影响但并非单调，hard-negative FPR 比总体 Macro-F1 更能揭示 motif 依赖；

第三，剪接识别信号可以转化为真实 ClinVar splice-altering vs benign 排序，full ClinVar 上 RNA-FM frozen encoder + MLP 表现最好，但 distance-matched 子集上的下降说明距离混杂必须被显式报告。

本文能够支持的核心结论是：真实 GT/AG hard-negative 是有效压力测试；真实小样本 ClinVar benchmark 可以用于变异效应排序诊断；distance-matched 分析能揭示近剪接距离捷径。本文不能支持的结论是：当前结果代表全基因组泛化、临床级变异解释、组织特异性剪接建模，或 alignment-level paralog/homology 去重后的稳健性能。
