\begin{abstract}
RNA 剪接是连接基因组序列、转录本结构与功能变异的重要分子过程。经典剪接位点通常包含 donor 侧的 GT 和 acceptor 侧的 AG 等局部共识序列，但真实剪接调控并不等价于对短 motif 的简单匹配：假阳性 GT/AG、上下游序列背景、组织使用差异以及单核苷酸变异都可能改变模型判断。围绕这一问题，本文基于当前仓库实现的三个实验构建一条完整的计算生物学分析链路：首先在小样本条件下建立 donor、acceptor 与 non-splice 三分类模型，验证基础剪接位点识别能力；随后通过多尺度上下文、hard-negative 和 rare motif 分析检验模型是否真正利用序列上下文；最后将识别模型转化为异常剪接变异效应预测器，并结合 zero-shot score、delta score、校准曲线和可解释性图件分析变异影响区域。

本项目不使用百万级完整数据集作为主实验输入，而采用可复现的小样本 split：训练集 855 条、验证集 120 条、测试集 285 条。实验一显示，RNABERT frozen/proxy 路线在测试集上达到 Macro-F1 0.9931，hard-negative FPR 为 0.0149；RNA-FM 路线也优于 CNN motif baseline。实验二显示，局部 motif 虽然提供强信号，但 CNN baseline 在 hard-negative 上仍有 0.4179 的误报率，而上下文模型与 splice signal proxy 明显降低误报。实验三进一步构造 460 条人工变异，覆盖 donor loss、acceptor loss、donor gain、acceptor gain 与 neutral far SNV，RNABERT zero-shot token distance 在人工变异排序上达到 AUROC 0.9990、AUPRC 0.9993。整体结果说明：在小样本、proxy/fallback 边界清晰的前提下，剪接识别、上下文验证和变异效应解释可以形成一条逻辑闭合的课程项目分析流程。
\end{abstract}

\begin{abstract*}
RNA splicing links genomic sequence, transcript structure, and functional variants. Canonical splice sites often contain local consensus motifs such as donor-side GT and acceptor-side AG, but splice regulation cannot be reduced to short-motif matching. False-positive GT/AG sites, surrounding sequence context, tissue-specific usage, and single-nucleotide variants may all affect splice prediction. This course project builds a coherent computational biology workflow from three implemented experiments: splice-site three-class classification, multi-scale context and hard-negative analysis, and aberrant splicing variant effect prediction.

The main experiments use a reproducible small split rather than a million-scale dataset: 855 training samples, 120 validation samples, and 285 test samples. Experiment 1 establishes donor/acceptor/non-splice recognition. Experiment 2 tests whether the models rely on contextual evidence beyond local motifs. Experiment 3 converts splice recognition into variant effect scoring with zero-shot scores, delta scores, calibration, and interpretability profiles. The results show that the small-sample pipeline is sufficient for a logically complete course project, provided that proxy/fallback models and optional real-tool case studies are clearly separated.
\end{abstract*}

\chapter{引言}

RNA 剪接决定了前体 mRNA 中外显子与内含子的连接方式，是从基因组编码到转录本功能的关键中间层。剪接位点识别通常依赖 donor 位点附近的 GT、acceptor 位点附近的 AG，以及多聚嘧啶区、分支点和上下游调控元件等更复杂的信息。对于计算生物学任务而言，一个自然的问题是：模型究竟是在学习真实的剪接上下文，还是仅仅在匹配短 motif。

这一问题在变异效应预测中更加重要。一个单核苷酸变异可能破坏原有 donor 或 acceptor，也可能在非剪接区域制造新的 cryptic splice signal。若模型只能识别局部 GT/AG，它可能在 hard-negative 上产生大量误报，也难以解释变异造成的剪接信号变化。因此，单独报告三分类准确率是不够的；更完整的分析应当包括上下文消融、困难负例、非经典 motif、变异 delta score 和可解释性分析。

本文将当前仓库中的三个实验组织为一条递进叙述线。实验一回答“能否识别剪接位点”；实验二回答“识别是否依赖有效上下文而非简单 motif”；实验三回答“这种识别能力能否迁移到异常剪接变异效应预测”。三者共同构成从序列识别到机制验证再到变异解释的闭环。

\section{模型与 proxy/fallback 边界}

实验中包含三类模型。第一类是可在当前环境直接训练的 CNN motif baseline。第二类是 RNA-FM 和 RNABERT 的 frozen-encoder 路线：当本地预训练权重存在时使用 frozen encoder，否则使用确定性 proxy/fallback embedding。第三类是 SpliceAI、Pangolin、MMSplice、MaxEntScan 等 optional real-tool wrapper；在缺少真实外部依赖时，结果以 proxy fallback 或 small case study 标注。本文所有结论均限定在当前小样本和上述边界内，不声称完成真实外部工具的大规模评测。

\section{评估指标}

三分类任务报告 Accuracy、Macro-F1、AUROC、AUPRC，并单独统计 hard-negative FPR。hard-negative FPR 的分母限定为测试集中 \codepath{label == 2} 且 \codepath{negative_type} 包含 hard 的样本。变异效应任务将 splice-altering 与 neutral 作为二分类排序问题，报告 AUROC、AUPRC、Top-k recall 和 enrichment；同时按变异类型统计 mean impact score，并用校准曲线和 delta profile 解释模型行为。

\chapter{实验一：剪接位点三分类}

\section{实验设计}

实验一面向 donor、acceptor 和 non-splice 三分类任务，目标不是单独追求分类分数，而是为后续上下文消融和变异效应预测提供一个可复现的剪接信号识别基线。数据采用前文定义的小样本 split：训练集 855 条、验证集 120 条、测试集 285 条，其中测试集包含 67 条带有 GT/AG 等局部 motif 的 hard-negative 样本。该设定可以同时检验模型对真实剪接位点的识别能力，以及其是否容易被局部 motif 负例误导。

比较模型包括四类：CNN baseline、RNA-FM frozen/proxy、RNABERT frozen/proxy 和 SpliceAI signal proxy。前两类 neural baseline 用于比较局部卷积表示与 foundation-model 表示的差异；SpliceAI 行当前是 signal proxy fallback，仅作为剪接信号启发式参照，不作为真实 SpliceAI 完整推理结果。评价指标包括 Accuracy、Macro-F1、Macro AUROC、Macro AUPRC，并额外报告 hard-negative FPR。

\section{结果}

表 \ref{tab:exp1-test} 展示测试集核心结果。RNABERT frozen/proxy 路线取得最高 Accuracy 和 Macro-F1，分别为 0.9930 和 0.9931，并将 hard-negative FPR 降至 0.0149。RNA-FM 的 Macro-F1 达到 0.9519，也明显高于 CNN baseline。CNN baseline 的总体 Macro-F1 为 0.9005，但 hard-negative FPR 达到 0.4179，说明较高的总体分类分数并不能掩盖其对 motif-like 负例的误报风险。

SpliceAI signal proxy 的 AUROC 和 AUPRC 分别为 0.9981 和 0.9962，但 Macro-F1 只有 0.8680，hard-negative FPR 也最高。这一结果说明：排序类指标较高并不必然意味着固定决策阈值下的三分类表现稳定；同时，该行仍应被解释为 proxy/fallback 结果，而不是外部 SpliceAI 工具的真实 benchmark。

\section{小结}

实验一说明，在小样本条件下，foundation-model 表示已经能够支撑较稳定的剪接位点三分类识别；但 CNN baseline 和 signal proxy 在 hard-negative 上的误报表明，局部 GT/AG motif 仍可能造成明显假阳性。因此，实验一的高分不能被孤立解读，必须继续通过窗口消融、困难负例和 rare motif 分析检验模型是否真正利用了更大范围的序列上下文。这一问题构成实验二的直接动机。

\section{小结}

实验二将实验一的分类结果推进到机制层面：局部 motif 是必要线索，但不是充分条件。hard-negative、rare motif 和 tissue/long-range case study 共同说明，剪接模型需要在局部共识序列之外利用更丰富的上下文信息。这为实验三中“变异如何改变剪接信号”提供了依据。

分类型结果显示，loss 类变异更容易被模型识别。例如 RNABERT zero-shot token distance 在 acceptor loss 与 donor loss 上的 mean score 均约为 1.75，而在 acceptor gain 和 donor gain 上分别为 0.2054 和 0.2712。SpliceAI signal proxy 对 donor loss 和 acceptor loss 也有明显响应，但对 gain 类变异响应较弱。这与生物学直觉一致：破坏已有强剪接位点通常比在背景序列中制造新剪接位点更容易检测。

\section{ClinVar、sQTL 与可解释性}

ClinVar smoke 仅包含 12 条变异，用于验证流程可运行，而不是完整 ClinVar benchmark。在该 smoke 集上，RNABERT zero-shot token distance 和 RNA-FM zero-shot embedding distance 均达到 AUROC 1.0000、AUPRC 1.0000；SpliceAI signal proxy 达到 AUROC 0.9167、AUPRC 0.9167。由于样本极少，这一结果只能说明流程闭环，不能作为泛化性能结论。

sQTL case study 保留 variant id、组织、目标基因、目标 junction、观察方向和模型 delta score 等字段，用于展示变异效应与组织剪接事件之间的连接方式。该部分同样不声称完成全量 GTEx 建模。

\section{小结}

实验三完成了从剪接识别到异常变异效应预测的迁移。zero-shot distance 对人工变异排序最稳定，loss 类变异比 gain 类变异更容易识别，ClinVar 与 sQTL case study 则展示了流程如何连接到真实资源。由于数据规模较小，所有真实资源相关结论均应视为 smoke 或 case study，而不是全量基准。

\chapter{讨论}

\section{三个实验之间的逻辑关系}

本文的核心并不是分别报告三个独立实验，而是建立一条递进链路。实验一提供基础识别能力；实验二检验这种能力是否建立在合理上下文之上；实验三将该能力转化为变异效应评分，并通过解释性分析回到序列机制。若没有实验二，实验一的高分可能只是 motif 记忆；若没有实验三，剪接识别只能停留在分类任务，无法服务于变异解释。

\section{小样本设定的意义与限制}

本项目明确不使用百万级完整数据集，而选择约千条级别的小样本 split。这一选择降低了存储和运行门槛，使课程项目能够完整覆盖数据构建、模型训练、结果评估和报告生成。小样本设置也迫使报告清晰区分主实验、proxy/fallback、optional real-tool wrapper 和 smoke case study。

限制同样明确。首先，小样本结果不能代表全基因组泛化性能。其次，部分 foundation model 和真实剪接工具当前依赖 proxy/fallback，不能写成完整真实模型评测。第三，ClinVar 和 GTEx/sQTL 只做少量 case study，不能得出临床或组织特异性总体结论。后续工作可在相同代码结构下接入本地权重、完整 raw data 和更大规模真实变异集。

\chapter{结论}

本文基于当前仓库实现，完成了一篇围绕 RNA 剪接的计算生物学课程项目论文。三个实验共同说明：第一，小样本条件下可以建立 donor、acceptor、non-splice 三分类模型；第二，hard-negative 和 rare motif 分析证明局部 motif 不是充分条件，模型需要利用上下文；第三，剪接识别模型可以进一步转化为异常剪接变异效应预测，并通过 zero-shot score、delta profile 和校准分析给出可解释结果。

因此，项目形成了从序列分类到机制验证再到变异解释的完整叙述线。其价值不在于宣称大规模 benchmark，而在于提供一个边界清晰、可复现、可扩展的课程项目范式。
