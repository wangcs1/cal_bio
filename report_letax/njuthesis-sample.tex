\begin{abstract}
RNA 剪接是连接基因组序列、转录本结构与功能变异的重要分子过程。经典剪接位点通常包含 donor 侧的 GT 和 acceptor 侧的 AG 等局部共识序列，但是，真实剪接调控并不等价于对短 motif 的简单匹配：假阳性 GT/AG、上下游序列背景、组织使用差异以及单核苷酸变异，都可能改变模型判断。围绕这一问题，本文设计并实现了三个实验，构建了一条完整的计算生物学分析链路：首先在小样本条件下建立 donor、acceptor 与 non-splice 三分类模型，验证基础剪接位点识别能力；随后通过多尺度上下文、hard-negative 和 rare motif 分析检验模型是否真正利用序列上下文；最后将识别模型转化为异常剪接变异效应预测器，并结合 zero-shot score、delta score、分箱响应曲线和可解释性图件分析变异影响区域。

本项目采用可复现的小样本 synthetic benchmark：训练集 855 条、验证集 120 条、测试集 285 条，并采用 chromosome-style synthetic 标签分组生成 split，质控报告显示 valid/test 与 train 无 gene overlap；但该划分不等同于真实基因组坐标上的 chromosome holdout。由于当前主实验默认不依赖外部预训练权重和完整真实工具，本文将 RNA-FM、RNABERT、SpliceAI、Pangolin 等名称严格解释为本仓库实现中的 frozen-or-fallback 路线或 signal proxy，而不是对应论文工具的全量 benchmark。实验一显示，RNABERT fallback token+signal 特征路线在测试集上达到 Macro-F1 0.9931，hard-negative FPR 为 0.0149；实验二显示，CNN motif baseline 在 hard-negative 上仍有 0.4179 的误报率，说明局部 GT/AG 不是充分条件；实验三构造 460 条人工变异，覆盖 donor loss、acceptor loss、donor gain、acceptor gain 与 neutral far SNV，zero-shot distance 在人工变异排序上达到接近饱和的 AUROC/AUPRC。该结果更适合作为流程闭环和 synthetic 机制验证，而不能直接外推为真实临床变异泛化能力。
\end{abstract}

\begin{abstract*}
RNA splicing links genomic sequence, transcript structure, and functional variants. Canonical splice sites often contain local consensus motifs such as donor-side GT and acceptor-side AG, but splice regulation cannot be reduced to short-motif matching. False-positive GT/AG sites, surrounding sequence context, tissue-specific usage, and single-nucleotide variants may all affect splice prediction. This course project builds a coherent computational biology workflow from three implemented experiments: splice-site three-class classification, multi-scale context and hard-negative analysis, and aberrant splicing variant effect prediction.

The main experiments use a reproducible small synthetic benchmark rather than a million-scale dataset: 855 training samples, 120 validation samples, and 285 test samples. The split follows a chromosome-style synthetic grouping rule rather than a real genomic chromosome holdout, and the QC report finds no gene overlap between train and valid/test. Experiment 1 establishes donor/acceptor/non-splice recognition. Experiment 2 tests whether the models rely on contextual evidence beyond local motifs. Experiment 3 converts splice recognition into variant effect scoring with zero-shot scores, delta scores, score-bin response curves, and interpretability profiles. The results support a logically complete course-project pipeline, but all claims are restricted to the current synthetic/proxy and smoke-test setting.
\end{abstract*}

\chapter{引言}

RNA 剪接是从基因组编码到转录本功能的关键中间层，决定了前体 mRNA 中外显子与内含子的连接方式。剪接位点识别通常依赖 donor 位点附近的 GT、acceptor 位点附近的 AG，以及多聚嘧啶区、分支点和上下游调控元件等信息。

已有研究已经从经典剪接信号、最大熵模型和深度神经网络等角度证明，剪接位点附近的序列包含强预测信号\cite{black2003,yeo2004,jaganathan2019}。近年来，SpliceAI、Pangolin、MMSplice 等工具进一步将序列模型用于变异效应预测\cite{jaganathan2019,cheng2019,zeng2022}；RNA-FM、RNABERT 等 RNA language model 则提供了从大规模无标注 RNA 序列中学习表示的思路\cite{chen2022rnafm,akiyama2022rnabert}。这些工作共同提示，剪接预测不应只被理解为简单的二核苷酸查找。

本文关注一个更适合课程项目规模验证的问题：剪接模型究竟是在学习真实的剪接上下文，还是仅仅在匹配短 motif？

这一问题在变异效应预测中同样重要，一个单核苷酸变异可能破坏原有 donor 或 acceptor，也可能在非剪接区域制造新的 cryptic splice signal，若模型只能识别局部 GT/AG，它可能在 hard-negative 上产生大量误报，也可能难以解释变异造成的剪接信号变化。

因此，单独报告三分类准确率是不够的，更完整的分析应当包括：1.上下文消融，2.困难负例，3.非经典 motif，4.变异 delta score 和可解释性分析。

对此，我们将全文组织为三个研究问题。

\begin{itemize}
  \item RQ1：在小样本条件下，模型能否区分 donor、acceptor 与 non-splice？
  \item RQ2：模型是否真正利用了局部 GT/AG motif 之外的上下文信息？
  \item RQ3：剪接识别信号能否转化为变异效应评分，并解释 gain/loss 类变异？
\end{itemize}

实验一回答 RQ1。实验二主要通过多尺度上下文、hard-negative 与 rare motif 回答 RQ2；组织使用与长程调控 case study 仅作为 pipeline 扩展示例。实验三通过人工变异、ClinVar smoke、sQTL case study 与解释性图件回答 RQ3。

\chapter{数据与方法}

\section{数据来源、构造与小样本设定}

项目主实验使用当前仓库默认的 synthetic splice benchmark，而不是全基因组真实数据集。该设置的目标不是复现 SpliceAI 或 Pangolin 等工具的大规模性能，而是在有限计算条件下构造可控的剪接识别、上下文验证与变异效应分析闭环。真实资源 \codepath{data/raw/genome.fa}、\codepath{data/raw/gencode.gtf}、ClinVar 和 GTEx/sQTL 文件在仓库中被标记为 optional smoke/case-study 资源，默认主结果不依赖它们。

synthetic benchmark 由 \codepath{src/data/build_synthetic_splice_dataset.py} 生成。每条序列先按指定 GC 范围采样随机 DNA 背景，再在中心附近插入类别相关信号：donor 样本包含中心 donor motif 及上下游短调控片段；acceptor 样本包含 AG、polypyrimidine tract 和 branch-point-like 片段；non-splice 样本分为 easy negative 与 hard negative，其中 hard negative 在中心附近插入 GT 或 AG，但标签仍为 non-splice。这个构造使得 GT/AG 出现本身不是充分判据，从而可以直接检验模型是否过度依赖短 motif。

所有主分类样本均保存为 DNA 字母 A/C/G/T；输入 RNA foundation model 路线时，代码会按模型需要将 T 转换为 U。主 split 采用 chromosome-style synthetic 标签规则：synthetic 样本被赋予 chr1--chr16、chr17--chr18、chr19--chr22/chrX 等分组标签，分别进入 train、valid 和 test。该规则用于避免同一 synthetic gene id 同时出现在 train 与 valid/test 中；\codepath{reports/data_qc.md} 对当前 split 检查到 valid/test 与 train 的 overlapping genes 为 0。但这不等同于真实基因组坐标上的 chromosome holdout，也不是同源序列聚类去重后的真实基因组泛化评估。

主实验序列长度为 401 bp，即中心位点左右各 200 bp。表 \ref{tab:data-split} 给出 split 规模。

\section{变异数据构造}

实验三使用 \codepath{src/data/build_variant_dataset.py} 从 test split 构造人工 SNV 变异集。donor loss 在 donor 样本中心 +1 位置破坏 GT；acceptor loss 在 acceptor 样本中心 -2 位置破坏 AG；donor gain 和 acceptor gain 在 non-splice 样本中寻找距离中心不超过 70 bp、且可通过 1 个碱基替换形成 GT 或 AG 的二核苷酸；neutral far SNV 从距中心约 90--180 bp 的远端位置随机替换碱基。构造后每个变异均保留 WT/Mut 序列、相对位置、目标类别、gene id 和 transcript id。

\codepath{data/experiment_3/artificial_variant_effect.csv} 包含 460 条人工变异，其中 splice-altering 280 条，neutral 180 条。ClinVar smoke 与 GTEx/sQTL case study 只用于验证输入输出流程和展示字段连接方式，不作为全局 benchmark。

\section{训练设置与复现细节}

所有可训练模型使用固定随机种子 42。实验一的 CNN baseline 使用 PyTorch Conv1D：三个卷积分支的 kernel size 分别为 3、5、9；每个分支包含 Conv1D、BatchNorm、ReLU、第二层 Conv1D、BatchNorm、ReLU 和 pooling；分类头使用 dropout、128 维全连接层、ReLU、dropout 与三分类输出层。CNN 训练 5 个 epoch，batch size 为 256，优化器为 AdamW，学习率 \(1\times10^{-3}\)，weight decay 为 \(1\times10^{-4}\)，损失函数为带类别权重的 cross entropy。当前实现报告 validation 指标，但不做 early stopping，保存最后一轮模型。

RNA-FM/RNABERT fallback 路线在缺少本地权重时使用确定性 k-mer、中心位置碱基和 engineered splice signal 特征；分类器为 scikit-learn \codepath{LogisticRegression}，\codepath{class_weight="balanced"}，\codepath{solver="lbfgs"}。实验一 frozen-or-fallback 路线使用 \codepath{C=1.5}、\codepath{max_iter=1200}；实验二/三的 lightweight fallback classifier 使用 \codepath{C=2.0}、\codepath{max_iter=1000}。SpliceAI-style signal proxy 使用 engineered signal 特征训练 RandomForest，\codepath{n_estimators=180}，\codepath{max_depth=9}，\codepath{min_samples_leaf=2}，并将随机森林概率与启发式 splice signal 概率按 0.72/0.28 混合。实验三的 zero-shot distance 和 pseudo-likelihood 分数不训练参数，只在 WT/Mut 序列之间计算确定性差异。

\section{评估指标}

三分类任务：评估 Accuracy、Macro-F1、AUROC、AUPRC，并单独统计 hard-negative FPR（hard-negative FPR 的分母限定为测试集中 \codepath{label == 2} 且 \codepath{negative_type} 包含 hard 的样本）。

变异效应任务：将 splice-altering 与 neutral 作为二分类排序问题，主表评估 AUROC 与 AUPRC；Top-k recall 和 enrichment 由于区分度有限，放入附录作为补充指标。同时，按变异类型统计 mean impact score，并用分箱响应曲线和 delta profile 解释模型行为。

\chapter{实验一：剪接位点三分类}

\section{实验设计}

实验一将序列分为 donor、acceptor 和 non-splice 三类，建立上下文分析和变异效应预测的基础分类器。

模型类别包括 CNN motif baseline、RNA-FM fallback k-mer+signal、RNABERT fallback token+signal 和 SpliceAI-style signal proxy。这里的 RNA-FM/RNABERT 名称表示本仓库 frozen-or-fallback 路线；在当前默认环境下，表中数值应按 fallback 特征路线解读。

训练使用 855 条样本，验证使用 120 条样本，测试使用 285 条样本。

\section{结果}

表 \ref{tab:exp1-test} 展示测试集结果，与基线进行对比。

RNABERT fallback token+signal 路线取得最高 Macro-F1，将 hard-negative FPR 降至了 0.0149；

RNA-FM 相比 CNN baseline 也有明显提升；

SpliceAI-style signal proxy 的 AUROC 和 AUPRC 较高，但 hard-negative FPR 仍高于 RNABERT fallback token+signal，因此不能简单等同于真实 SpliceAI 完整推理结论。

\section{小结}

实验一说明，在小样本条件下，序列模型已经能够完成基础剪接位点识别。然而， CNN baseline 在 hard-negative 上误报较高，说明模型可能过度依赖局部 motif。

这个问题引出实验二：剪接识别是否需要更大范围的上下文证据？

\chapter{实验二：多尺度上下文与困难负例}

\section{实验设计}

实验二围绕三个问题展开：

第一，改变上下文窗口大小 （pm50，pm100，pm200，pm400），观察模型性能是否随上下文变化。

第二，构造 GT/AG hard-negative，检验模型是否把局部 motif 误判为真实剪接位点。

第三，将 tissue usage 和 long-range case study 作为 pipeline 扩展示例，展示当前框架如何连接组织使用字段和长程调控输出，但不把它们作为回答 RQ2 的强证据。

\section{多尺度上下文结果}

表 \ref{tab:exp2-context} 给出了不同窗口下的 Macro-F1 与 hard-negative FPR。

CNN motif baseline 的 Macro-F1 稳定在 0.8733，hard-negative FPR 维持在 0.4179，说明模型对窗口变化不敏感。

相比之下，RNABERT fallback token+signal、RNA-FM fallback k-mer+signal 和 SpliceAI-style signal proxy 在多个窗口上的 hard-negative 误报都显著降低。但这些结果不支持“上下文越长越好”的简单结论：RNABERT fallback token+signal 在 pm50 和 pm100 上达到 1.0000，而在 pm200、pm400 上分别下降到 0.9895 和 0.9825；RNA-FM fallback k-mer+signal 也存在非单调变化。更合理的解释是，hard-negative 比普通测试集更能揭示 motif 依赖问题，而不同窗口长度会改变噪声、局部信号密度和训练/测试分布。

\section{小结}

实验二将实验一的分类结果推进到机制层面：局部 motif 是必要线索，但不能保证分类正确。

实验二主要由 hard-negative 与 rare motif 压力测试支撑。更准确地说，本文能支持的结论是：在当前 synthetic/proxy 设置中，单独依赖中心 GT/AG 会导致可观误报；加入位置、k-mer、engineered signal 或更明确的 splice signal 后，模型更容易拒绝 motif-matched negative。组织使用与长程调控 case study 展示了 pipeline 扩展方向，但不构成 GTEx 或长程模型基准。这为实验三中“变异如何改变剪接信号”提供了依据。

\section{ClinVar、sQTL 与可解释性}

\section{小结}

实验三完成了从剪接识别到异常变异效应预测的进一步深入，得出 zero-shot distance 对人工变异排序最稳定、loss 类变异比 gain 类变异更容易识别的结论。

但这一结论应限制在人工变异与 proxy/fallback 设置内。ClinVar 与 sQTL case study 展示了流程闭环，由于数据规模较小，相关结论应视为 smoke 或 case study，而不是全量基准。

\chapter{讨论}

\section{三个实验之间的逻辑关系}

本文的三个实验构建了一条递进链路。实验一提供基础识别能力；实验二检验这种能力是否建立在合理上下文之上；实验三将该能力转化为变异效应评分，并通过解释性分析解读序列机制。

若没有实验二，实验一的高分可能只是 motif 记忆，若没有实验三，剪接识别只能停留在分类任务，无法应用到变异解释。

\section{关键现象解释}

第一，CNN baseline 在 hard-negative 上失败，主要原因并不是模型完全不能学习剪接，而是当前 hard-negative 被有意构造成“携带 GT/AG 但标签为 non-splice”。卷积模型能抓住中心附近强 motif，因此普通测试样本上分数不低；但当负例也带有相同二核苷酸时，缺少更明确的位置、上下游 signal 或较长范围统计特征会导致误报。这说明 hard-negative 是比总体 Accuracy 更敏感的诊断指标。

第二，zero-shot distance 在人工变异上优于分类器迁移，并不一定意味着无监督模型本质上更适合临床变异预测。更直接的解释是，人工变异的标签由序列扰动规则定义，而距离型分数正好度量 WT 与 Mut 表示的变化幅度；分类器则是在 donor/acceptor/non-splice 标签上训练，其输出概率未必对单碱基扰动具有良好的局部连续性。因此，distance score 更适合作为当前 synthetic 扰动的排序量。

第三，loss 类变异比 gain 类变异更容易识别，既有生物学原因，也有构造原因。生物学上，破坏已有强 donor/acceptor 通常会造成明显 signal loss；构造上，loss 变异直接作用于中心强 motif，而 gain 变异需要在 non-splice 背景中制造新 motif，背景上下文未必支持该 motif 成为稳定剪接信号。因此，gain 类结果应结合 delta profile 个案解释，而不宜只看均值。

\section{小样本设定的意义与限制}

本项目没有使用完整数据集，而选择约千条级别的小样本 split。这降低了存储和运行门槛，使本项目能够完整覆盖数据构建，模型训练，结果评估和报告生成。

限制同样明显：首先，小样本 synthetic 结果不能代表全基因组泛化性能；其次，当前 split 虽然避免 train/test gene overlap，但没有证明跨物种、跨注释版本或同源序列聚类后的泛化；第三，部分 foundation model 和真实剪接工具当前依赖 proxy/fallback，不能作为完整模型评测；第四，ClinVar 和 GTEx/sQTL 只做少量 case study，不能得出组织特异性总体结论。

后续工作可在当前代码基础上接入 GENCODE/GRCh38 构建器\cite{frankish2019}、更严格的 gene/chromosome/homology split、真实 ClinVar/SpliceVarDB 风格变异集、以及本地真实 RNA-FM/RNABERT/SpliceAI/Pangolin 权重或官方工具输出。只有在这些条件满足后，才适合将本文结果升级为真实 benchmark 结论。

\chapter{结论}

本项目构建了序列分类→机制验证→变异解释的完整链路。

三个实验共同说明：

第一，小样本条件下可以建立 donor、acceptor、non-splice 三分类模型；

第二，hard-negative 和 rare motif 分析表明，在当前 synthetic/proxy 设置中，局部 motif 不是充分判据；模型若仅依赖 GT/AG，容易在 motif-matched negative 上产生误报；

第三，剪接识别模型可以进一步转化为异常剪接变异效应预测，并通过 zero-shot score、delta profile 和分箱响应分析给出可解释结果。

本文能够支持的核心结论是：在当前小样本 synthetic/proxy 条件下，hard-negative 能有效揭示 motif 依赖，zero-shot distance 能捕捉人工剪接变异扰动，解释性图件能把标量分数连接回序列位置。本文不能支持的结论是：fallback 结果等同于真实 RNA-FM/RNABERT/SpliceAI/Pangolin benchmark，或 12 条 ClinVar smoke 可以代表临床变异泛化能力。
